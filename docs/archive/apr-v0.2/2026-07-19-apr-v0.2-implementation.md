# Android Project Runtime v0.2 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build the TypeScript/Node.js APR CLI that records and deterministically verifies Android Journeys through Gradle, Android CLI, and ADB.

**Architecture:** Keep Journey and report models as pure domain code. Application services orchestrate injected ports; production adapters wrap child processes, while tests use deterministic fakes. The CLI exposes human-readable output and a stable machine-readable `apr verify --json` contract.

**Tech Stack:** Node.js, TypeScript ESM, npm, Commander, Inquirer, Zod, Vitest, ESLint, Android CLI, ADB, Gradle Wrapper

---

## Execution rules

- Work only in the `codex/apr-v0.2` worktree.
- Follow @superpowers:test-driven-development for every production behavior.
- Run the narrow test first, then the full suite before each commit.
- Do not call Android CLI or ADB from domain/application modules; use ports.
- Do not use shell command strings. Pass executable and argument arrays to the process runner.
- Keep `apr verify --json` stdout machine-only; diagnostics go to stderr.
